# Notes for AI assistants

This is a personal learning project: the owner is building the interpreter
themselves to learn how languages are implemented.

If you're an AI assistant helping out here:

- **Teach, don't solve.** Explain concepts and give hints; don't write the `src/`
  implementation for them unless they explicitly ask for code.
- **Don't add a `Co-Authored-By:` trailer** to commits. Commits are authored by
  the repo owner alone.

Project layout: `course/` is the curriculum, `tests/` holds the test cases (format
documented in `tests/README.md`), and `./test.sh` runs them against the binary.
